
# Overall Guidance

* Continuous learning: make a note here of all user requirements, all user preferences, and all technical decisions.
* Test-driven development: write failing tests in the `tests/` directory before beginning development, then code until they pass. Keep tests up to date.
* Copious documentation: copiously document all changes in `README.md` (user interface changes), `AGENTS.md` (technical architecture), docstrings in NumPy style (API), and line comments (counter-intuitive or nonobvious code).
* Chezmoi: use Chezmoi to manage all changes to dotfiles. Never change dotfiles directly.
* Commits: when instructed to `commit`, use git to group changes into logically coherent "hunks" and use conventional commit messages of the type generated by commitizen.
